# TheLawyer

> Multi-tenant practice management for US small law firms (5–25 attorneys). Matter intake → time → billing → trust-account reconciliation, with an AI assistant that answers questions about any matter on demand.

Generated by [`abrahamfernandez/the-workflow`](https://github.com/abrahamfernandez/the-workflow) from the legal industry research artifact at [`research/legal.md`](research/legal.md). The full chain — research → spec → plan → architecture → composed `workflow.json` → generated code — is checked into this repo.

## Status

- **Foundations epic — skeleton complete.** Solution + projects compile; Aspire AppHost composes Postgres (with pgvector) + Redis + the API + the Vite dev server; SPA shell renders with sidebar/topbar layout; Terraform skeleton for Azure; CI builds backend + web + validates Terraform.
- **Subsequent epics (Matters, Documents, Calendar, Billing, Trust, Portal, AI Assistant, Connectors, Reporting) are not yet generated.** See [`PLAN.md`](PLAN.md) for the build order.

## Getting started

Prerequisites: .NET 10 SDK, Docker (for Aspire-managed Postgres + Redis), Node.js 22+, optionally Azure CLI + Terraform 1.9 for IaC.

```bash
# Backend + frontend + dependencies all start together via Aspire.
dotnet run --project src/TheLawyer.AppHost

# Then in the Aspire dashboard URL printed in the console:
#   - api          → the .NET API on a random port
#   - web          → Vite dev server (default :5173)
#   - postgres     → managed Postgres with pgvector
#   - redis        → managed Redis
#   - pgadmin      → web UI for poking at the DB
```

Health endpoints:

```bash
curl http://localhost:<api-port>/health     # 200 OK
curl http://localhost:<api-port>/api/v1/ping
```

## Stack at a glance

| Concern | Choice |
|---|---|
| Backend | .NET 10 + Aspire + minimal APIs |
| Frontend | Vite + React 18 + TypeScript + shadcn/ui + Tailwind + PWA |
| Persistence | PostgreSQL 16 + EF Core 9 |
| Vector store | pgvector (same Postgres instance, behind `IVectorStore`) |
| Cache + idempotency | Redis 7 |
| Background jobs | Hangfire (Postgres-backed) |
| AuthN | Azure AD B2C (single tenant for staff + clients) |
| AI | Anthropic Claude via private endpoint with no-retention contract |
| IaC | Terraform 1.9 |
| Observability | OpenTelemetry → Azure Monitor in prod; Aspire dashboard locally |
| CI/CD | GitHub Actions |

## How this repo was generated

TheLawyer was scaffolded end-to-end using [TheWorkflow](https://github.com/abrahamfernandez/the-workflow) — an agentic Claude Code plugin that walks Claude through every phase of building an enterprise system. Each artifact in this repo is the output of a phase:

| Phase | Skill | Output |
|---|---|---|
| 1. Bootstrap | `init-system` | `workflow.json`, `.claude/settings.json`, LICENSE, .gitignore |
| 2. Research | `research-industry` | `research/legal.md` |
| 3. Spec | `synthesize-spec` | `SPEC.md` |
| 4. Plan | `plan-system` | `PLAN.md` |
| 5. Design | `design-architecture` | `ARCH.md`, `docs/diagrams/c[1-3]-*.puml`, `DECISIONS.md` |
| 6. Compose | `manage-workflow` + `manage-skills` | Updated `workflow.json` + `.claude/settings.json` |
| 7. Build | `build-system` | The Foundations epic (everything else generates in later iterations) |
